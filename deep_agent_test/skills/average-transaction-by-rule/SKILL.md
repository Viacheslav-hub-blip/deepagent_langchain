# Расчет средней суммы транзакции по правилу

Используй этот skill, когда пользователь просит рассчитать среднюю, минимальную, максимальную сумму или другую статистику по транзакциям для сработок с конкретным правилом антифрода.

## Примеры запросов

```text
Средняя сумма транзакции у сработок с правилом 'DENY оплата обучения после смены устройства'.
Посчитай минимальную и максимальную сумму по сработкам с правилом 'RULE_NAME'.
Какая средняя сумма по сработкам с правилом, содержащим 'обучения'?
```

## Шаги

1. **Поиск сработок по правилу.**
   - Вызови `task(data-retrieval-agent)` для поиска в таблице hits (`cspfs_repo_features3.hits_extra_info_129372427_view`).
   - Фильтр: `main_rule` содержит название правила.
   - Если правило хранится в JSON-формате, используй поиск по подстроке JSON.
   - Запроси поля: `event_id`, `transaction_amount`, `transaction_amount_in_rub`, `event_dt`, `epk_id`, `main_rule`.
   - Период: без ограничения, если пользователь не указал.

2. **Проверка результата.**
   - Если сработки найдены — переходи к шагу 3.
   - Если сработок нет — проверь точное название правила через выборку уникальных значений `main_rule`, содержащих ключевые слова из запроса.

3. **Расчет статистики.**
   - Извлеки значения `transaction_amount` или `transaction_amount_in_rub` из preview_rows отчёта data-retrieval-agent.
   - Вызови `execute_python_code` для расчета:
     - Средняя сумма: `np.mean(amounts)`
     - Количество сработок: `len(amounts)`
     - Минимальная сумма: `min(amounts)`
     - Максимальная сумма: `max(amounts)`
   - При большом количестве строк (>100) читай полный набор из pickle-файла через `read_pickle_file`.

4. **Формирование ответа.**
   - Укажи найденное количество сработок.
   - Укажи рассчитанную среднюю сумму (и другую запрошенную статистику).
   - Укажи период сработок (минимальная и максимальная `event_dt`).
   - Укажи название правила.

## Шаблон финального ответа

```text
Вывод:
Средняя сумма транзакции по сработкам с правилом '<RULE_NAME>' составляет <AVG_AMOUNT> руб.

Детали:
- Количество сработок: <N>
- Период сработок: с <MIN_DT> по <MAX_DT>
- Минимальная сумма: <MIN_AMOUNT> руб.
- Максимальная сумма: <MAX_AMOUNT> руб.

Использованные источники:
- Таблица: cspfs_repo_features3.hits_extra_info_129372427_view
- Поля: event_id, transaction_amount, transaction_amount_in_rub, event_dt, main_rule
- Фильтр: main_rule содержит '<RULE_NAME>'
```

## Ограничения

- Поле `main_rule` может хранить правило в JSON-формате — используй поиск по подстроке.
- Если сработок не найдено, проверь точное название правила через уникальные значения `main_rule`.
- Для больших выборок (>100 строк) читай данные из pickle-файла, а не из preview.
