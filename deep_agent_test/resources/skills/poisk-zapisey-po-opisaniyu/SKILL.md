# Поиск записей по смысловому описанию

Отбор по смыслу строится на текстовом поле `event_description`. Структурированные поля (коды операций, MCC, merchant-признаки) — только доп. контекст для проверки и разбивок, не замена отбору.

## Примеры запросов

```text
Найди сработки, связанные с оплатой доставки за январь 2025 года.
Посчитай операции, похожие на оплату такси.
Выгрузи записи, относящиеся к подпискам.
Найди события по оплате маркетплейсов.
Сколько было операций, связанных с доставкой еды?
```

## Шаги

1. **Источник и поля.** Выбери таблицу с `event_description` и нужным периодом. Только поля, подтверждённые в schema / context / skill / tool output.
2. **Все строки `event_description` за период.** `load_data` со `select_columns="event_description"` и фильтром по периоду. Без агрегации, без ограничивающего `max_rows`.
3. **Уникальные описания и смысловой отбор.** Из полного набора (шаг 2) выведи уникальные `event_description` через `execute_python_code` (`drop_duplicates`/`value_counts`). Выбери ВСЕ значения по смыслу: синонимы, формы, близкие формулировки (для «образование» — «оплата обучения», «онлайн-курсов», «образовательной платформы», «экзамена»). Спорные — включай как кандидатов.
4. **Полная выгрузка.** `load_data` за тот же период с фильтром `event_description in [<значения из шага 3>]` и нужными полями.
5. **Подсчёт.** В `execute_python_code` посчитай всего и по разбивкам (по `event_description`, при наличии — по кодам операций / MCC / мерчантам).

## Ограничения

- На шаге 2 не агрегируй и не схлопывай строки, не задавай ограничивающий `max_rows` — нужны все строки.
- Если результат ушёл в offload-файл (pkl), дочитай его целиком через `execute_python_code` (`read_pickle_file`). Не делай выводов по preview.
- Уникальные значения выводи кодом из полного набора, а не через `group_by` на шаге 2.
- Не отбирай по случайному вхождению слова и не по неполному сэмплу.
