# Поиск записей по смысловому описанию

Используй, когда пользователь просит найти, посчитать, выгрузить или проанализировать операции/события по смысловой категории.

Базовая таблица: обычно `hits`, если пользователь говорит о сработках. Для raw-истории переходи в `cards` или `uko` по правилам `/skills/hit-table/joins.md`.

## Алгоритм

1. Выбери таблицу с `event_description` и нужным периодом.
2. Загрузи полный набор строк `event_description` за период без агрегации и без ограничивающего `max_rows`.
3. Через `execute_python_code` найди уникальные `event_description` из полного набора.
4. Выбери все значения, которые по смыслу соответствуют категории пользователя.
5. Выполни полную выгрузку по выбранным `event_description`.
6. Посчитай результат через `execute_python_code`, особенно если есть `.pkl`.

## Пример для load_data

```text
table_name: hits
select_columns: ["event_description", "event_dt"]
filters:
  - {"column": "event_dt", "operator": "between", "values": ["20260101", "20260131"]}
```

## Ограничения

- Не делай выводы по preview.
- Не используй `group_by` для первого шага уникальных описаний: сначала нужен полный набор.
- Если результат сохранён в `.pkl`, дальше обрабатывай его через `execute_python_code`, без повторного `load_data`.
- Для категории "образование" включай близкие формулировки: обучение, онлайн-курсы, образовательная платформа, экзамен, репетитор, учебные материалы.

## Когда читать дополнительные файлы

- Если нужна таблица `hits`, смотри `/skills/hit-table/SKILL.md`.
- Если нужны raw-поля карточного канала, смотри `/skills/cards-event-table/SKILL.md`.
- Если нужны raw-поля ДБО/СБП, смотри `/skills/uko-event-table/SKILL.md`.
