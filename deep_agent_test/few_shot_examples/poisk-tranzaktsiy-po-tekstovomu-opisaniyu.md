name: Поиск транзакций связанных с оплатой доставки по описанию

Пример описывает кейс, когда пользователь просит найти события, транзакции или сработки,
связанные с заданной темой по текстовому описанию операции. Например, нужно найти
транзакции, относящиеся к оплате доставки, по полям description, merchant_name, mcc_code
и дате события. Пример полезен для задач, где сначала нужно выгрузить широкий набор
кандидатов за период, сохранить его в файл, затем семантически проанализировать текстовые
описания и выбрать записи, которые действительно относятся к заданной категории.

---

## Пользовательский запрос

Найди транзакции, связанные с оплатой доставки, по текстовому описанию операций.

## Примерный план работы

1. Уточнить у пользователя период выборки, целевую категорию поиска и названия таблиц, если они не указаны явно.
2. Через data-retrieval-agent сделать выборку всех релевантных полей за указанный промежуток времени: description, mcc_code, event_dt, merchant_name и идентификатор записи.
3. Сохранить широкую выборку кандидатов в файл, чтобы дальнейший анализ опирался на зафиксированный набор данных.
4. Проанализировать все значения description, merchant_name и mcc_code загрузив их в контекст, выбрать записи на основе смысла (семантики), которые относятся к оплате доставки по смыслу текстового описания, и определить id найденных записей.
5. Сделать отдельную выборку найденных id из сохраненного файла или из исходной таблицы
6. Сохранить записи, связанные с оплатой доставки, в отдельный файл.
После последнего инструментального пункта assistant возвращает пользователю ответ с описанием найденных транзакций: количество записей, период, использованные таблицы и поля, критерии отбора, путь к файлу и ограничения анализа.

## Важные правила

- описания в description могут быть написаны по разному, поэтому искать записи по вхождению слов - неправильно, 
необходимо выбрать нужные записи с помощью LLM
