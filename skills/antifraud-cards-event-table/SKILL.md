# Карточные события: `csp_afpc_sss_inc.cards_event`

Все карточные события клиента (включая без срабатывания АФ).

**Когда:** карточные покупки, POS, E-commerce, ATM, авторизации, проверка карты, токенизация, NFC — по `event_channel` / `surface` / `product` из сработки или по смыслу задачи.

**Связь с hits:** в первую очередь `event_id` в обеих таблицах; иначе клиент + дата/время + сумма + тип (+ карточный id при наличии).

**Не путать:** строка в `cards_event` не означает сработку; событие может не быть в hits, если правило не сработало. Не использовать для чисто UKO/не карточных сценариев.
