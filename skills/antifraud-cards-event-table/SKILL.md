# Карточные события: `csp_afpc_sss_inc.cards_event`

Все карточные события клиента (включая без срабатывания АФ).

**Алиасы таблицы:** `cards_event`, `csp_afpc_sss_inc.cards_event`, `cspfs_repo_features3.cards_event`.

**Ключевые поля схемы:**
- `event_id` — идентификатор события, используется для связи с `hits`.
- `epk_id` — ЕПК/PPRB идентификатор клиента.
- `user_id` — внутренний идентификатор клиента в raw-таблицах.
- `event_dt` — дата события в формате `YYYYMMDD`.
- `event_time` — дата и время события.
- `event_channel` — канал карточного события, например `ISSUER`.
- `transaction_amount` — сумма операции в исходной валюте.
- `transaction_amount_in_rub` — сумма операции в рублях.
- `type_operation` — тип операции, например `CARD_PURCHASE`.

**Когда:** карточные покупки, POS, E-commerce, ATM, авторизации, проверка карты, токенизация, NFC — по `event_channel` / `surface` / `product` из сработки или по смыслу задачи.

**Связь с hits:** в первую очередь `event_id` в обеих таблицах; иначе клиент + дата/время + сумма + тип (+ карточный id при наличии).

**Не путать:** строка в `cards_event` не означает сработку; событие может не быть в hits, если правило не сработало. Не использовать для чисто UKO/не карточных сценариев.
