# Transaction Pattern Investigation

Описание: точка входа для расследования поведения вокруг сработки.

Открой:

- `../antifraud-hit-table/SKILL.md` - сработка и ключи;
- `../_shared/data-sources.md` - выбор raw-таблицы;
- `../_shared/antifraud-core.md` - ограничения выводов.

Порядок:

1. Зафиксируй `event_id` или `epk_id + event_dt`, канал, сумму, тип, получателя/ТСТ, `main_rule`, `policy_action`, резолюцию.
2. Построй окно 180 дней до текущей сработки включительно.
3. Сравни прошлые сработки по частоте, каналам, правилам, суммам и контрагентам.
4. Для нужных дней выгрузи полные `cards_event` или `uko_event`.
5. Ищи похожие операции: тот же получатель/merchant, близкая сумма, канал; затем ослабляй критерии.
6. Раздели вывод на долгосрочный паттерн и сценарий внутри дня.

Если нет `epk_id` или `event_dt`, явно укажи, что историческое окно построить нельзя.
