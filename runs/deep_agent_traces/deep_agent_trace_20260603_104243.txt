Описание: универсальная безопасная выборка из доступных Spark-таблиц по короткому alias. Инструмент принимает один параметр query: SQL-подобный текст запроса. Агент пишет query по skills, а внутренний нормализатор преобразует его в структурированные аргументы и выполняет выборку. При успешной выборке возвращается pandas DataFrame с полным результатом запроса.

Когда использовать:
- нужно прочитать строки, события или агрегаты из таблиц hits, cards, uko, history_automarking или demo_client_timeline;
- известны таблица, период, нужные колонки и фильтры по ключам/значениям;
- нужно проверить наличие записей, получить фактические поля события или посчитать агрегат по данным источника.

Когда не использовать:
- нет периода, даты начала или даты конца: сначала запроси недостающие данные;
- нужно обработать уже выгруженный pickle/offload-файл: используй код поверх сохраненного результата, а не повторный load_data;
- нужна произвольная Spark SQL-команда, join нескольких источников, запись данных, удаление данных или изменение таблиц;
- требуется SELECT * / SELECT all: перечисли только нужные колонки.

Параметры:
- query (str, обяз.): SQL-подобный запрос. В query обязательно укажи LOAD/FROM с коротким alias, PERIOD или date BETWEEN, SELECT с явными колонками или агрегатами, при необходимости WHERE/GROUP BY/ORDER BY/LIMIT.

Формат query:
  LOAD <table_alias>
  PERIOD <date_column> FROM '<YYYYMMDD>' TO '<YYYYMMDD>'
  SELECT <column_1>, <column_2> [, COUNT(*) AS <alias>] [, count(<column>) AS <alias>]
  WHERE <column> = '<value>' AND (<column> LIKE '%value%' OR <column> CONTAINS '<value>')
  GROUP BY <column>
  ORDER BY <column> ASC|DESC
  LIMIT <int>

Допустимые таблицы: hits, cards, uko, history_automarking, demo_client_timeline. Вместо LOAD можно использовать FROM, но имя источника должно быть коротким alias, а не Spark-путем, именем файла, saved_file, virtual_file или pkl.

Операторы WHERE:
- равенство: =, ==, eq, equals -> внутренне нормализуется в eq;
- не равно: !=, <>, ne, not_equals -> ne;
- сравнения: >, >=, <, <=, gt, gte, lt, lte;
- текстовый поиск: LIKE '%value%' или CONTAINS 'value' -> contains;
- списки и интервалы: IN (...), BETWEEN <from> AND <to>;
- несколько условий можно соединять через AND и OR.

Ограничения:
- период обязателен для каждого запроса и задается через PERIOD или WHERE <date_column> BETWEEN '<from>' AND '<to>';
- SELECT * и SELECT all запрещены для обычной выборки, но COUNT(*) разрешен в агрегатах;
- длинные идентификаторы передавай строками в кавычках, чтобы не потерять точность.

Обычный select возвращает строки как есть, БЕЗ устранения дублей: уникальные значения выводи сам по полному набору.

Операторы WHERE:
- равенство: =, ==, eq, equals -> внутренне нормализуется в eq;
- не равно: !=, <>, ne, not_equals -> ne;
- сравнения: >, >=, <, <=, gt, gte, lt, lte;
- текстовый поиск: LIKE '%value%' или CONTAINS 'value' -> contains;
- списки и интервалы: IN (...), BETWEEN <from> AND <to>;
- несколько условий можно соединять через AND и OR.

Ограничения:
- период обязателен для каждого запроса и задается через PERIOD или WHERE <date_column> BETWEEN '<from>' AND '<to>';

- длинные идентификаторы передавай строками в кавычках, чтобы не потерять точность.
